## 🧠 핵심 개념 (Deep Dive)
해당 챕터에서 다루는 기술적 디테일과 작동 원리를 상세히 기록합니다. 정확한 이해를 위해 필요한 아키텍처나 매개변수의 역할 등을 정리하기 좋은 공간입니다.

### 1. LLM
*   **정의 및 역할**: LLM은 방대한 텍스트로 학습한 크고 범용적인 언어 모델이다. 책이나 기사, 포럼 등 다양한 자료에서 나타난 텍스트의 패턴을 학습해 일반적인 텍스트 작업(텍스트 생성, 요약, 번역, 분류 등)을 수행한다.
*   **기존 방식과의 차이점**: 특정 작업에 특화된 머신러닝 방법들과 달리 매우 범용적으로 사용할 수 있다.

### 2. Transformer Neural Network Architecture
*   **정의 및 역할**: 문장 내의 각 단어와 다른 모든 단어의 관계를 고려해 문맥을 파악하며, 이를 토대로 연속체에서 나타날 가장 가능성이 높은 다음 단어를 예측한다. 
*   **주요 파라미터 / 메서드**:
    *   `invoke()`: 단일 입력에 대한 동기적 실행
    *   `batch()`: 여러 입력에 대한 병렬 처리 실행 (처리 속도 향상 목적)
*   **기존 방식과의 차이점**: [이전 기술이나 다른 방법론과 비교했을 때의 장단점 또는 성능 차이]

### 3. Prompt
*   **정의 및 역할**: LLM에 제공하는 지침과 입력 테스트를 의미한다. 프롬프트는 LLM의 출력 품질에 많은 영향을 미치며 좋은 출력을 만드는 프롬프트를 작성하는 과정을 **Prompt Design** 혹은 **Prompt Engineering**이라고 한다.

### 4. Pre-Trained LLM
*   **정의 및 역할**: 지도 학습을 통해 사전에 학습된 LLM을 의미한다. 
*   **지도 학습 기법**:
    *   `다음 단어 예측`: 학습 데이터의 각 문장에서 마지막 단어를 제거해 입력과 예상 출력 쌍을 만든다.
    *   `누락 단어 예측`: 각 문장에서 중간 단어 하나를 생략해 입력과 예상 출력의 다른 쌍을 만든다.
*   **기존 방식과의 차이점**: 기존 머신러닝 모델의 지도 학습과 달리 LLM의 지도 학습은 자동으로 학습 데이터에서 입력과 예상 출력을 구성하여 학습한다.

## 🛠️ 기술 사양 및 파이프라인 구조
구현한 체인(Chain)이나 모델의 데이터 흐름(Data Flow)을 도식화하거나 구조적으로 설명합니다.

*   **사용 모델**: (예: `gpt-4o`, `claude-3-5-sonnet`) - *해당 태스크에 이 모델을 선택한 이유(예: 추론 능력, 속도 등) 기록*
*   **파이프라인 흐름**:
    1. `PromptTemplate`: 사용자 입력을 특정 형식으로 포맷팅
    2. `LLM`: 포맷팅된 프롬프트를 처리하여 응답 생성
    3. `OutputParser`: 생성된 텍스트 응답을 JSON 구조로 파싱 및 검증
